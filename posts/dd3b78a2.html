组件

vue组件的基本使用

创建一个根组件，在vue3中组件就是一个普通的js对象，组件用来创建组件实例，组件是组件实例的模板

组件 –》组件生成组件实例 –》虚拟DOM –》DOM（在页面中呈现）

组件只是用来创建组件实例（vm）的，因为组件要进行复用。，且data要返回一个对象而不是直接等于一个对象，也是要进一步对组件进行复用，根据不同的组件实例，返回不同的对象

创建template的三种方式：

  • 在组件中通过template创建
  • 直接在网页的根元素中创建
  • 在组件中通过render()直接渲染

插值语法{{}}：只能填写表达式（有返回值的）

单文件形式的组件

template是用字符串的形式在编写模板

  • 这些字符串会在项目运行时，在浏览器中被编译为js的函数（性能不太好）
  • 在字符串中编写代码，体验很差

为解决这个问题，vue提供了一种单文件组件（SRC）

  • 单文件组件的格式是vue
  • vue文件用来编写单文件组件，vue文件本身并不能被浏览器所识别，因此它需要被构建工具打包后才能使用，在打包的过程中会被打包工具转换成js函数，避免了在浏览器中进行编译

组件生命周期

组件生命周期从setup开始在vue3中setup代替了vue2中的beforeCreate，和created。

  1. setup：此阶段开始初始化实例，props解析，计算属性，响应式数据，并设置各种init Options Api
  2. onBeforeMount：此阶段准备渲染dom
  3. onMounted：dom渲染完毕，组件挂载完毕（不包括异步组件）
  4. onBeforeUpdate：在组件因响应式更新之前访问dom状态，在这个钩子中更改状态也是安全的。
  5. onUpdate：在组件因响应式更新后调用
  6. onBeforeUnmount：在组件实例被销毁之前调用，此时组件实例依然用全部的功能
  7. onRenderTracked：收集响应式依赖
  8. onRenderTriggered： 触发响应式依赖

响应式原理

代理对象

vm.$data是实际的代理对象，通过vm可以直接访问到$data中的属性，通过vm.$data可以动态的向组件中添加响应式数据，但是不建议这么做，容易造成数据混乱，一般还是通过对象返回

组件间通信

父传子

子组件中的数据不会通常不会在子组件中直接定义，这样会导致数据和视图产生耦合，子组件中的数据一般会在创建组件实例时确定，父组件可以通过defineProps来将数据传递给子组件

使用：

  • 在子组件中定义props：defineProps()（宏指令）

注意：

  • 父组件传递子组件的数据是单向流动的，对子组件来说，父组件传递的数据只是可读的，不能进行修改，保障了数据的安全性（大部分框架，修改数据的地方一般只能存在于某一个组件中，不会存在多个组件都可以对数据进行修改）

props的配置：

在ts中使用defineProps：

子传父

使用：

  • 在子组件中定义emit：通过defineProps给父组件传值

暴露

通过defineExpose来暴露组件中的值或方法

provide

在父组件中定义的值通过provide（依赖注入）可以在所有的子，孙组件中通过inject读取使用

参数：

  1. key：定义注入的值的名字，通过inject读取时要使用key来读取
  2. value：注入的值，可以通过readonly来设置只读，禁止子组件修改

兄弟组件传值

  • 方式一：通过父组件作为中间过渡（子—父—子）

方式二：通过js的发布订阅模式

方式三：通过插件mitt（原理还是发布订阅模式）

各种组件功能

全局组件

如果一个组件使用频率非常高，就可以把这个组件定义为全局组件。

使用：

  • 全局组件需要在入口文件main.js中进行引入
  • 通过根组件App.vue的实例的，方法component定义全局组件

递归组件

当一个组件要接收未知数量的值时，可以将该组件定义成递归组件

用法：

  • 确定递归组件的名字
    1. 在vue3中可以直接使用文件名来当做递归组件的名字
    2. 在创建一个script标签，来自定义递归组件的名字
    3. 使用第三方插件

在递归组件中，可以通过$event来传递事件对象，递归组件会触发冒泡，可以通过事件修饰符stop来阻止冒泡

动态组件

component是一个动态组件，通过is属性来决定最终以什么标签呈现。

使用动态组件（component）进行切换时，会造成一些不必要的性能浪费 如：

Vue received a Component which was made a reactive object.

This can lead to unnecessary performance overhead,

and should be avoided by marking the component with markRaw or using shallowRef instead of ref.

Vue接收到一个组件，该组件是一个响应式对象。这可能会导致不必要的性能开销，

应该通过使用’ markRaw ‘标记组件或使用’ shallowRef ‘而不是’ ref ‘来避免。

优化：

  • markRaw: 跳过跟踪，会有一个__v_skip属性

  • shallowRef: 浅层响应，深层对象不会被proxy代理

在第二个script的标签中，通过components来注册组件，组件名通过字符串的形式书写，就不用再用markRaw来跳过追踪了。

异步组件

使用：

  • 使用axios来请求组件数据

  • 异步组件通过defineAsyncComponent来进行引入，它接收一个回调函数，以import函数引入异步组件路径

  • 使用Suspense标签来使用异步组件，它包含两个插槽，#default用于展示异步组件。#fallback用于展示异步组件还没响应时的内容。

传送组件

Teleport：组件可以将指定的组件传送到任意的dom节点。

属性：

  • to：必选，指定传送的dom节点
  • disabled：默认值：false，为true时to属性失效

缓存组件

keep-alive：缓存组件实例，使其不被卸载、销毁。

使用：

  • 使用keep-alive标签包裹组件，组件会自动缓存
  • 属性：
    1. include：只缓存指定的组件
    2. exclude：不缓存指定的组件
    3. max：最多缓存几个组件，超出这个数量，会对不活跃，比较旧的组件进行卸载

使用keep-alive，会新增两个生命周期：onActivated，onDeactivated，

  • onActivated：当缓存的组件重新被使用，则会触发该生命周期，表示keep-alive激活，此时不会再触发onMount（只会在一开始触发一次），当组件不在使用，则触发onDeactivated。
  • onDeactivated：当组件不在使用，则触发onDeactivated，此时不会触发onUnmount，因为keep-alive已经对组件进行了缓存，当组件重新使用则触发onActivated

过渡组件

transition过渡组件，用于添加元素的过渡效果

属性：

  • name：定义css类名，默认为：v
  • appear：在页面加载时，执行一次动画，切换时不会执行
  • duration：定义过渡的时间，可以写成对象的形式，为enter和leave分别设置过渡时间，也可以写成字符串，统一定义过渡时间
  • enter，leavecss类名的别
  • tag：定义transiton组件中的内容，要被哪种元素包裹

生命周期：

transition组件的生命周期有八个：

  1. @before-enter：进入之前，被插入Dom之前
  2. @enter：被插入Dom后开始执行动画效果，参数done函数，会在动画完成时执行
  3. @after-enter：过渡结束
  4. @enter-cancelled：过渡动画被打断
  5. @before-leave：离开之前，元素被隐藏或删除时执行
  6. @leave：离开动画
  7. @after-leave：离开完成，只会在元素从Dom中删除时（v-if）执行，（v-show）隐藏则不会执行
  8. @leave-cancelled：离开动画被打断，只会在元素被隐藏时（v-show）执行，删除则不会

可以配合一些动画库进行切换如：https://animate.style/，https://greensock.com/。

transition-group

用于列表（循环遍历）的过渡，属性与transition一样

平面过渡：数组中元素位置发生变化的过渡

状态过渡：一个数字从旧值变化到新值的过渡

插槽

使用：

  • 通过slot标签作为插槽内容的出口，v-slot指令作为标识，（可简写为#，匿名插槽要简写成#default）

默认插槽

在组件的标签之间没有任何内容时，默认插槽会为组件填充一些默认的内容来显示，当组件标签之间有内容时，默认插槽的内容会被覆盖

具名插槽

除了默认插槽外，slot标签还可以添加一个name属性配合v-slot指令，来指定特定的插槽出口

匿名插槽

匿名插槽是没有指定name属性的具名插槽，当一个组件同时接收默认插槽和具名插槽时，所有位于顶级的没有被<template> 标签包裹的，都会被隐式地视为默认插槽的内容。

插槽作用域

动态插槽

动态插槽就是为v-slot指定一个变量，可以动态的指定插槽的出口
