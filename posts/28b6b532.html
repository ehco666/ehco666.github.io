Node 核心模块

核心模块

process

核心模块,是 node 中自带的模块,可以在 node 中直接使用

  • global是 node 中的全局对象,类似于 window,ES 标准下全局作用的标准名是globalThis

核心模块:

process:表示当前的 node 进程,通过该对象可以获取进程信息,或者对进程做各种操作

属性和方法:

  • process.exit():结束当前进程,终止 node,可以传递终止的状态码

  • process.nextTick():将函数插入到 tick 队列中

    此时代码的执行顺序: 调用栈>tick 队列>微任务队列>宏任务队列

  • tick 队列中的代码,会在下一次事件循环之前执行,在微任务与宏任务队列中的任务之前执行,tick 队列在早期用来代替微任务队列

path,fs

path:表示路径,通过 path 可以用来获取各种路径,要使用 path 要先进行引入

方法:

  • path.resolve():用来生成一个绝对路径
    • 相对路径: ./xxx ../xxx xxx
    • 绝对路径:
      • 在计算机本地:windows:c:\xxx ,linux:/User/xxx
      • 在网络中:http://www.xxx/..., https://www.xxx/...
    • 直接调用 resolve 会返回当前工作目录,且通过不同方式执行代码时,获取的工作目录会不同
    • 将一个相对路径作为参数时,会转换为绝对路径,此时根据工作目录的不同,产生的绝对路径也不同
    • 为防止产生不同的结果,一般第一个参数传递绝对路径(即__dirname),第二个参数传递相对路径

fs(File System):fs 用来帮助 node 来操作磁盘中的文件(即 I/O 操作),使用 fs 前也需要进行引入

方法:

  • fs.readFileSync():同步的读取文件的方法,会阻塞后面代码的执行
    • 当我们通过 fs 模块读取磁盘中的数据时,读取到的数据总会以Buffer对象的形式返回
    • Buffer是从磁盘读取到内存时的缓冲区,用来临时存储数据
  • fsp.appendFile():创建新文件或者将数据添加到已有文件中
  • fs.mkdir():创建目录
  • fs.rmkdir():删除目录
  • fs.rm():创建文件
  • fs.rename():重命名
  • fs.copyFile():复制文件
