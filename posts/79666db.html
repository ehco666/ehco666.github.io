数据结构

栈

介绍

  • 栈:(stack)又名堆栈，它是一种运算受限的线性表。限定仅在表尾进行插入和删除操作的线性表

  • 特点: 后进先出(LIFO: Last In First Out)

队列

一般队列

介绍

  • 队列(Queue):

    队列是一种特殊的线性表，特殊之处在于它只允许在表的前端（front）进行删除操作，

    而在表的后端（rear）进行插入操作，和栈一样，队列是一种操作受限制的线性表。

  • 特点: 先进先出(FIFO: First In First Out)

链表

单向链表

介绍

  • 链表是一种物理存储单元上非连续、非顺序的存储结构，数据元素的逻辑顺序是通过链表中的指针链接次序实现的。
  • 链表由一系列节点（链表中每一个元素称为节点）组成，节点可以在运行时动态生成。
  • 每个节点包括两个部分：一个是存储数据元素的数据域，另一个是存储下一个结点地址的指针域.
  • 优点：
    1. 相比于线性表顺序结构，操作复杂。由于不必须按顺序存储，链表在插入的时候可以达到O(1)的复杂度，比另一种线性表顺序表快得多。
    2. 使用链表结构可以克服数组需要预先知道数据大小的缺点，链表结构可以充分利用计算机内存空间，实现灵活的内存动态管理。
  • 缺点：
    1. 但是查找一个节点或者访问特定编号的节点则需要O(n)的时间，而线性表和顺序表相应的时间复杂度分别是O(logn)和O(1)。
    2. 但是链表失去了数组随机读取的优点，同时链表由于增加了结点的指针域，空间开销比较大

双向链表

介绍

  • 在单向链表中，一个节点只有对下一个节点的引用，查找数据只能从头遍历到尾，或者从尾遍历到头，在达到中间某个节点时，不可以回到该节点的前一个节点，只能继续往下走。

    在插入或删除数据时，只需考虑两个引用

  • 在双向链表中，一个节点同时保存了对下一个节点的引用和上一个节点的引用，在到达某个节点时，可以返回前一个节点，也可以继续前往下一个节点，灵活性相对较高。

    在插入或删除数据时，要考虑到四个引用，相比于单向链表，占用的内存也会多一点。

哈希表

介绍

  • 哈希表是一种使用数组作为底层存储结构的数据结构，但又类似字典或集合，数据的存储是无序且不重复的。，它的特点是可以在接近O(1)的时间内进行读写操作。哈希表的核心是哈希函数，它可以把任意类型的键转换成一个整数，作为数组的下标。

哈希表：

  • 哈希化
    • 将字符串转成较大数字（hashcode），幂的连乘，底数一般要为质数，然后通过霍纳算法进行优化减少乘法
    • 再把hashcode压缩到数组的范围之内
  • 哈希函数
    • 哈希化后封装的函数，哈希函数一定会产生冲突；
    • 解决冲突
      • 链地址法
        • 对象的下标位置，不在单纯的存储一个值，而是存放一个数组或者链表（因为采用线性查找，数组与链表的性能都差不多，但如果要数据要插入到开头建议使用链表，要插入到结尾使用链表或者数组都可以）
      • 开放地址法
        • 使用线性探测，寻找空白的位置进行插入，插入的数据可能不会均匀的分布空白位置，不同的元素映射到同一个位置，会产生数据聚集的问题
        • 解决数据聚集
          • 二次探测
          • 在哈希法
      • 性能对比
        • 插入数据，会使填充因子变大，填充因子越来越大，探测步长会不断增大，效率会不断下降。
        • 开放地址法，中填充因子最大为1，如果填充因子过大，效率会急剧下降
        • 链地址法，中理论上没有最大值，但随着填充因子的逐步增大，效率也会逐步下降，但是比较稳定
        • 优秀的哈希函数，中要减少乘法和除法的使用，遵循霍纳法则，使元素在哈希表中均匀的分布

树

一般数结构

  • 在查找数据时，数组需要进行线性查找，效率是O(n)。而哈希表查找数据的效率是O(1)，非常的快，但是哈希表容易产生冲突，空间利用率不高，且数据的存储时无序的，不可以按固定的顺序进行遍历，只能通过key来访问数据。但如果对一个数组进行排序在进行二分查找，那查找的效率会使O(logn)，这样的查找方式也可以看成是一种树结构。
  • 树结构（Tree）
    • 树是由n（n>=0）个节点构成的有限集合。当n=0时称为空树。当n>0时，会有一个根节点root，其他节点可以分为m（m>0）个互不相交的有限集合，而每个集合本身又是一个树（称为子树）
      1. 节点的度（Degree）：节点的子树的个数
      2. 树的度：树的所有节点中最大的节点度数
      3. 叶子节点（Leaf）：度为0的节点
      4. 父节点（Parent）：有子树的节点
      5. 子节点（Child）：父节点中的节点
      6. 兄弟节点（Sibling）：有同一个父节点
      7. 路径和路径长度：路径：从节点n到k多经过的节点个数，路径长度：路径所包含的边的个数，（等于，路径-1）。
      8. 节点的层次（Level）：规定根节点在1层，其他节点的层数，是父节点+1。
      9. 树的深度（Depth）：树中所有节点中的最大层次，是树的深度

二叉树

  • 树中每个节点最多只能有两个子节点，几乎所以的树都可以转化成二叉树。
    1. 一个二叉树第i层的最大节点数为：2^(i-1), i>=1。
    2. 深度为k的二叉树有最大节点总数为：2^k-1, k>=1。
    3. 对任何非空二叉树T，若n0表示叶子节点的个数，n2表示度为2的非叶子节点的个数，那么两者的关系满足：n0=n2+1。
  • 完美二叉树（Perfect Binary Tree）
    • 除了最下层次的叶子节点外，每层的节点都有2个子节点
  • 完全二叉树（Complete Binary Tree）
    • 除了最下层次的叶子节点外，每层的节点都有2个子节点
    • 最下层次的叶子节点连续存在，只能在右侧有所缺失
    • 完美二叉树是特殊的完全二叉树

二叉搜索树（BST）

介绍

  • 二叉搜索树（Binary Search Tree），是一颗二叉树可以为空
  • 若不为空则必须满足以下性质：
    1. 非空左子树的所有键值小于根节点的键值
    2. 非空右子树的所有键值大于根节点的键值
    3. 左，右子树本身都是二叉搜索树
