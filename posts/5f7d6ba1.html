两道面试题

网络基础

Http协议

网络的服务器基于请求(request)和响应(response),请求和响应实际上就是一段数据,只是这段数据需要遵循特殊的格式,

这个特殊的格式由HTTP决定(请求报文与响应报文)

https://github.com/ehco666

https://:协议名 使用那种协议进行请求,列如:http,ftp等

github.com:域名,(domain),整个网络中有数不清的服务器,每一个服务器都有自己的唯一标识,这个标识被称为ip地址,列如:192.168.1.17,但是ip地址不方便记忆,因此域名就相当于ip地址的别称

/ehco666:网站资源的路径,表示你要访问网站中的某个资源

TCP/IP 协议族

  • TCP/IP协议族中包含了一组协议，这道协议规定了互联网中所有的通信的细节
  • 网络通信的过程有四层组成
    1. 应用层(http等协议)
      • 软件的层面，浏览器，服务器都属于应用层
    2. 传输层(tcp等协议)
      • 负责对数据进行拆分，把大数据拆分成一个一个的小包
    3. 网络层(ip等协议)
      • 负责给数据包添加信息
    4. 数据链路层(一大堆协议)
      • 传输数据包
  • http协议就是应用层的协议,用来规定客户端与服务器间通信的报文格式

报文(message)

请求报文(request):

  • 请求报文格式:

    1. 请求首行,请求报文的第一行,(列如:GET /index.html?username=admin&password=123456 HTTP/1.1)

      • 第一部分GET

        get表示请求的方式,常见的方式为get和post请求,get请求用来向服务器请求资源,post请求用来向服务器发送数据

      • 第二部分index.html?username=admin&password=123456

        表示请求资源的路径,?后面的内容叫做查询字符串,查询字符串是一个名值对结构(一个名字对应一个值),使用=链接,多个名值对之间使用&分割

        get请求:
        通过查询字符串将数据发送给服务器,由于查询字符串会在浏览器地址栏中显示出来,所以安全性较差,同时由于url地址长度有限制,所以get请求无法发送较大的数据

        post请求:

        通过请求体来发送数据,在chrome中可以通过载荷查看,无法通过地址栏查看,安全性较好,且请求体没有大小限制,尽量使用post向服务器发送数据

      • 第三部分HTTP/1.1

        http协议的版本

    2. 请求头

      • 请求头也是名值对结构,用来告诉服务器我们浏览器的信息,每一个请求头都有他的作用

      • Accept 浏览器可以接受的文件类型(列如:text/html)

      • Accept-Encoding浏览器允许的压缩编码

      • User-Agent用户代理,他是用来描述浏览器信息的字符串

    3. 空行

      • 用来分割请求头和请求体

    4. 请求体

      • post请求通过请求体来发送数据

响应报文(response):

  • 响应报文格式:

    1. 响应首行

      • HTTP/1.1 200 OK

        HTTP/1.1协议版本

        200响应状态码

        ok对响应状态码的描述

        响应状态码的规则:

        1xx请求处理中

        2xx表示成功

        3xx表示请求的重定向

        4xx客户端错误

        5xx服务器错误

    2. 响应头

      响应头也是一个一个的名值对结构,用来告诉浏览器响应的信息

      Content-Type:用来描述响应体的类型

      Content-Length:用来描述响应体的大小

    3. 空行

      分割响应头与响应体

    4. 响应体

      网页主体(body)

服务器的主要功能:

  • 可以接收到浏览器发送的请求报文
  • 可以向浏览器返回响应报文

1.当在浏览器中输入地址以后发生了什么?

  1. DNS解析,获取网站的IP地址(获取网站的真实地址)

  2. 浏览器需要和服务器建立链接(tcp/ip)(三次握手)

    客户端如何与服务器建立(断开)链接(tcp)

    • 通过三次握手建立链接,通过四次挥手断开链接
    • 三次握手
      1. 客户端向服务器发送连接请求(SYN–同步标识)
      2. 服务器收到连接请求,向客户端返回消息(消息:1:同意链接(ACK–同意标识),消息2:申请与客户端建立连接(SYN)
      3. 客户端向服务器发送同意连接的信息(ACK)
    • 四次挥手(断开连接时,不一定为客户端进行断开)
      1. 客户端向服务器发送请求,通知服务器数据发送完毕(FIN–完成标识),请求断开链接
      2. 服务器向客户端返回数据,得知客户端发送完毕(ACK)
      3. 服务器向客户端返回数据,表示数据接受完毕(FIN),同意断开连接(ACK)
      4. 客户端向服务器发送数据,同意断开连接(ACK)

  3. 向服务器发送请求(http协议)

  4. 服务器处理请求,并返回响应(http协议)

  5. 浏览器将响应的页面渲染

  6. 断开和服务器的连接(四次挥手)
