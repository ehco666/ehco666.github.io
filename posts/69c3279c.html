Git

git的简介

git:版本控制工具

git的基本配置及命令

git的基本配置:

git config --global user.name "name":配置git的用户名

git config --global user.email "email":配置git的用户邮箱

git config --global user.password "password":配置git的用户密码

git的常用命令:

git status:查看当前仓库的状态

git init:初始化仓库

git add xxx:暂存文件,xxx可以是指定的文件路径也可以是.表示全部文件(未跟踪–>暂存)

git add *:将所有已修改的文件暂存

git commit -m "xxx":提交暂存文件,xxx提交代码的日志信息,不能省略(暂存–>未修改)

git commit -a -m "xxx":提交所有已修改的文件

git restore <filename>:重置文件,恢复到上一次操作

git restore --staged <filename>:取消暂存状态

git rm <filename>:删除文件

git rm <filename> -f:强制删除

git mv form<filename> to<filename>:移动文件,重命名文件

文件状态

git的文件有两种状态:已跟踪(Tracked)和未跟踪(untracked),已跟踪的文件被git所管理,未跟踪的文件不受git管理

已跟踪的文件又有三种状态:

暂存(Staging):表示文件修改已经保存,但是尚未提交到git仓库中

已修改(Modified):表示本地磁盘中的文件已修改与git仓库中的文件不同

未修改(untouched):表示本地磁盘中的文件与git仓库中的文件相同,没有被修改

文件状态可以通过git status查看

分支

git在储存文件时,每一次提交代码都会创建一个对应的分支,git节点就是通过一个一个的节点记录代码的状态.节点会构成一个树状结构,树状结构会存在分支(一般默认为master),在使用git时,可以创建多个分支,分支与分支之间相互独立,互不影响.

git branch:查看当前分支

git branch <分支名>:创建一个新的分支

git branch -d <分支名>:删除指定分支

git branch -M <分支名>:修改分支名

git branch -a:查看所有分支

git switch <分支名>:切换到指定分支 (旧版为:check out)

git switch -c <分支名>:创建并切换到指定分支

git merge <分支名>:合并指定分支(Fast-forward:快速合并)

变基(rebase)

通过变基合并分支,使用git merge <分支名>合并分支时,在提交记录中会显示所有的分支创建和分支合并的全部过程,当项目比较复杂时,会使代码的提交记录变得非常混乱

git rebase <分支名>:变成指定分支名的基底

原理(变基时发生了什么?):

  • 当我们发起变基时,git会首先把两条分支的最近的共同祖先基底
  • 对比当前分支相对于祖先的历史提交,并且将他们提取出来存储到一个临时文件中
  • 将当前分支指向目标分支的基地
  • 以当前基底开始,重新执行历史代码的提交操作

变基和merge对于合并分支来说最终结果是一样的,但使用变基会使代码的提交记录更加清晰整洁.

远程仓库(remote)

将本地库上传到远程仓库

git remote add <remote name> <url>:使用url链接远程仓库

git push -u <remote name> <分支名>:提交代码到,远程仓库的指定分支并关联远程库(关联以后,可以直接使用git push进行提交)

git push:推送代码到已关联的分支

git push <remote name> <本地分支>:<远程分支>:指定本地分支并推送到指定的远程分支

常用操作命令:

git remote:查看关联的远程库名字

git remote -v:查看关联远程库的详细信息(名字,地址<url/ssh>)

git remote remove <remote name>:删除指定的远程库

注意:如果本第库的版本低于远程库的版本,git push默认是推不上去的,想要推送成功必须确保本地库与远程库的版本一致

git fetch:拉取远程库代码到本地库,拉取后必须使用git merge进行分支合并

git pull拉取远程库代码到本地库并自动合并分支

标签(tag)

当头指针(head)没有指向某个分支的头部时,称为分离头指针(head detached),分离头指针的状态下也可以操作代码,但不会出现任何的分支上,所以不要在分离头指针的状态下来操作仓库.

  • 通过git log可以常看当前的头指针,及每次提交的代码id
  • 使用git switch <提交代码id> --detached 来改变头指针的位置(如果切换后,头指针没有指向任何的分支,就是分离头指针的操作)

如果要回退到之前的节点对代码进行操作,则可以选择创建分支后在改变头指针的位置

  • 使用git switch -c <分支名> <提交代码id>来创建一个新的分支,并切换头指针

可以为提交记录设置标签,设置标签以后,可以通过标签快速的识别出不同的开发节点

  • git tag <标签名>:为当前的分支打上标签
  • git tag <标签名> <提交代码id>:为指定的节点打上标签(如果该节点不存在任何的分支,通过标签切换后依然会存在分离头指针的问题)
  • git tag:查看所有的标签

gitignore

  • 默认情况下,git会监视项目中所有内容,但是我们有事希望一些文件不要被git所管理(列如:node_modules,项目日志等),我们可以在项目目录中添加.gitignore文件,来设置需要被git所忽略的文件
